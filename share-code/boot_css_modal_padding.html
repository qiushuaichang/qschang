两个方法同时监听同一个对象

这是稍微偏浏览器底层的，两个方法有可能会同时监听到这一个对象，然后“资源竞争”，影响到其他的方面或者整体的样式。这样可以用“锁”去解决。原理就是执行一个方法的时候，就false另一个方法(进行判断，让这个方法不执行) 

            .box{
                flex-direction: row;    /*决定子元素排列方向*/
                flex-wrap: nowrap;      /*决定子元素排不下时是否换行*/
                /*
                justify-content : 子元素样式
                  flex-start：左对齐
        　　　　　　flex-end：右对齐
        　　　　　　center：居中对齐
        　　　　　　space-between：两端对齐
        　　　　　　space-around：沿轴线均匀分布
                */
                justify-content: space-around;
                /*
                align-items : 子元素样式(横主轴为例)
                  flex-start：顶端对齐
            　　　　flex-end：底部对齐
            　　　　center：竖直方向上居中对齐
            　　　　baseline：子元素第一行文字的底部对齐
            　　　　stretch：当子元素未设置高度时，子元素将和父容器等高对齐
                */
                align-items: flex-start;
                align-content: flex-start;  /*设置这个属性，align-items就不管用了*/
            }
            .box>div{
                height: 100px;
                margin: 10px 20px;
                order: 0;       /*子元素的排列顺序，越小排的越靠前*/
                flex-grow: 1;       /*父元素空间充足时，子元素扩大的比例*/
                flex-shrink: 1;     /*父元素空间不足时，子元素缩小的比例*/
                flex-basis: 10px;   /*决定子元素在子轴上占据的空间*/
                align-self: flex-start;     /*同align-items，可覆盖align-items属性*/
            }
